# Craig Johnson Portfolio - Go + Templ + HTMX + Tailwind

A modern, responsive personal portfolio website built with **Go**, **Templ**, **HTMX**, and **Tailwind CSS** for server-side rendering with dynamic client interactions and modern utility-first styling.

## Overview

A server-rendered Go application using Templ for type-safe component-based templates, HTMX for dynamic interactions, and Tailwind CSS v4 for modern utility-first styling. The site maintains a professional design and functionality while leveraging server-side rendering for improved performance and SEO.

## Features

- **Type-Safe Templates**: Templ provides compile-time type checking and component-based architecture
- **Server-Side Rendering**: Fast initial page loads with zero JavaScript dependencies
- **HTMX Interactions**: Dynamic content loading without full page refreshes
- **Tailwind CSS v4**: Modern utility-first CSS with CSS-first configuration
- **Responsive Design**: Mobile-first approach with beautiful desktop layouts
- **Professional UI**: Modern design with smooth animations and gradient effects
- **Soccer Tool**: HTMX-powered schedule fetcher with ICS download

## Pages

- **Home** - Hero section with profile, stats, and quick links
- **About** - Personal background and values
- **Experience** - Career timeline with HTMX lazy loading
- **Skills** - Technical proficiencies by category
- **Projects** - Project showcase with links
- **Education** - Academic background and certifications
- **Contact** - Professional contact information
- **Soccer** - Interactive schedule download tool

## Tech Stack

- **Backend**: Go 1.23+
- **Templating**: [Templ](https://templ.guide/) - Type-safe Go templating engine
- **Frontend Interactivity**: HTMX 1.9
- **Styling**: [Tailwind CSS v4](https://tailwindcss.com/) - Utility-first CSS framework
- **Fonts**: Inter (Google Fonts)

## Getting Started

### Prerequisites

- Go 1.23 or higher
- Templ CLI (installed automatically via `go install`)
- curl (for downloading Tailwind CSS binary)

### Installation

```bash
# Install Go dependencies
go mod download

# Install Templ CLI
go install github.com/a-h/templ/cmd/templ@v0.3.977

# Build the project (downloads Tailwind CSS, generates Templ components, and compiles)
make build
```

The Makefile will automatically download the standalone Tailwind CSS v4.1.18 binary to `bin/tailwindcss` on first build.

### Running

```bash
# Run the server
./portfolio-server

# Or use make
make run
```

The server will start at `http://localhost:8080`

### Development

For development with hot reload:

```bash
# Install air for Go hot reload
go install github.com/air-verse/air@latest

# Terminal 1: Start CSS watcher (auto-rebuilds on changes)
make watch-css

# Terminal 2: Start Go server with hot reload
make dev
```

The server will start at `http://localhost:8080`

**Note**: 
- When Templ files (`*.templ`) are modified, run `make generate` or `templ generate` to regenerate the Go code
- The standalone Tailwind CSS binary (116MB) is downloaded automatically on first build
- Tailwind scans all `.templ` files for utility classes and generates optimized CSS

## Design Principles

1. **Type-Safe Components**: Templ provides compile-time type checking for templates
2. **Utility-First CSS**: Tailwind CSS v4 for rapid UI development with consistency
3. **Progressive Enhancement**: Core content works without JavaScript
4. **HTMX for Interactivity**: Dynamic updates without SPA complexity
5. **Server-Rendered**: Fast initial loads, great SEO
6. **Mobile-First**: Responsive design starting from mobile
7. **Accessible**: Semantic HTML, ARIA labels, keyboard navigation
8. **Modern Styling**: Gradient effects, smooth transitions, and card-based layouts

## Tailwind CSS Configuration

This project uses **Tailwind CSS v4** with CSS-first configuration (no `tailwind.config.js` required).

### Theme Customization

Edit `static/css/tailwind.css` to customize the theme:

```css
@theme inline {
  /* Custom colors */
  --color-bg-primary: #1c1917;
  --color-accent-primary: #ea580c;
  
  /* Custom spacing, typography, etc. */
}
```

### Build Commands

```bash
# Download Tailwind CSS binary (done automatically on first build)
make install-tailwind

# Build CSS once
make build-css

# Watch for changes (development)
make watch-css

# Build everything (CSS + Templ + Go)
make build

# Clean build artifacts
make clean
```

The standalone Tailwind CSS binary is automatically downloaded from the [official GitHub releases](https://github.com/tailwindlabs/tailwindcss/releases/tag/v4.1.18) and cached in `bin/tailwindcss`.

### Adding Tailwind Classes

Simply add utility classes to your `.templ` files:

```templ
<div class="bg-[#292524] border border-white/[0.06] rounded-2xl p-8">
  <h2 class="text-2xl font-bold text-[#fb923c]">Hello Tailwind!</h2>
</div>
```

Tailwind automatically scans all `.templ` files and generates only the CSS you use.

## Customization

### Updating Content

Content is defined in `main.go` in the data functions:

- `experienceData()` - Work experience entries
- `skillsData()` - Skills by category
- `projectsData()` - Project showcase
- `educationData()` - Education entries

### Updating Templates

Templates are written in Templ (`.templ` files) with Tailwind utility classes:

1. Edit the `.templ` files in `components/` directory
2. Run `make generate` or `templ generate` to regenerate Go code
3. Tailwind automatically detects new classes when building CSS
4. Build and run: `make build && ./portfolio-server`

For more information on Templ syntax, see the [Templ documentation](https://templ.guide/).

### Styling with Tailwind

All styling is done with Tailwind utility classes directly in `.templ` files. No separate CSS files needed per page.

**Example patterns:**

```templ
<!-- Cards -->
<div class="bg-[#292524] border border-white/[0.06] rounded-2xl p-8">
  <!-- Card content -->
</div>

<!-- Buttons -->
<button class="px-8 py-4 bg-gradient-to-br from-[#ea580c] to-[#fb923c] text-white rounded-xl hover:-translate-y-0.5 transition-all">
  Click Me
</button>

<!-- Tech Pills -->
<span class="inline-flex items-center px-3 py-1 bg-[rgba(234,88,12,0.1)] text-[#fb923c] rounded-full border border-[rgba(234,88,12,0.2)]">
  Go
</span>
```

Theme colors and custom values are defined in `static/css/tailwind.css`.

## Deployment

The application is designed to be deployed as a standalone binary:

```bash
# Download Tailwind CSS and build CSS
make install-tailwind
make build-css

# Generate Templ components
templ generate

# Build for production
CGO_ENABLED=0 GOOS=linux go build -o portfolio-server .

# Run
./portfolio-server
```

For containerized deployment, create a Dockerfile:

```dockerfile
FROM golang:1.23-alpine AS builder
WORKDIR /app

# Install dependencies
RUN apk add --no-cache curl

# Install Templ CLI
RUN go install github.com/a-h/templ/cmd/templ@v0.3.977

# Copy source
COPY . .

# Download Tailwind CSS and build CSS
RUN make install-tailwind && make build-css

# Generate Templ components and build Go binary
RUN templ generate && CGO_ENABLED=0 go build -o portfolio-server .

FROM alpine:latest
WORKDIR /app
COPY --from=builder /app/portfolio-server .
COPY --from=builder /app/static ./static
EXPOSE 8080
CMD ["./portfolio-server"]
```

**Note**: The standalone Tailwind CSS binary is downloaded during the build process. Templ components are compiled into the Go binary, so only the `static/` directory and binary are needed in production.
